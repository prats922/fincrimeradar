# FinCrimeRadar Backlog Board

Three loops, rotating. Each working session picks one loop, clears or advances what's in it, then moves to the next. Don't run more than one loop per session, that's how things end up half finished.

Suggested cadence: fortnightly rotation, Polish → Build → Content → repeat. Polish goes first because it's the smallest and the most likely to get skipped if it's not first.

## 📝 Content Loop
Knowledge Hub articles and guide series. Ship two to three parts of a series close together, then move to the next topic rather than leaving series unfinished.

**Standing requirement, applies to every new guide from here on, confirmed by user, do not deviate without explicit sign-off:** guides must match the interactivity depth of kyc-onboarding-dilemma.html, not the lighter prose-plus-cheat-sheet format used in MLRO Handbook and Crypto Guide. Required per guide: multiple worked scenarios, each with a structure or relationship diagram where the content has one to show, a decision point per scenario with graded multiple-choice options and full reasoning revealed after the analyst chooses, and a multi-question knowledge check with live scoring at the end.

The end-of-guide visual summary is locked to a specific shape, finalized after prototyping 3 options: a reference table, one row per key pattern or category, three columns, "The trap" (why it looks clean or routine), "The tell" (the actual red flag combination), "Response" (the correct action). Icon plus category name in the first cell. Forest green brand system, Tabler-style icons throughout, colors mapped to brand.css custom properties, never hardcoded hex. Never modeled on generic AI-style corporate infographic templates, an explicitly rejected reference example had visible typos and an off-brand palette. **Required as part of the pattern, not optional:** the table must carry class `ref-table`, matched by `brand.js`'s table-scroll wrapper (alongside the existing `data-table`/`compare-table` selectors) so it's automatically wrapped in a scrollable `.table-scroll` container. Found missing on the shipped Fraud Red Flags guide: without it, "The tell" and "Response" columns overflowed the mobile viewport with no way to reach them. `brand.css`'s nowrap-on-scroll rule for this wrapper is itself scoped to `max-width:760px` so it only kicks in where content actually needs it, this table's longer sentence-style cells wrap normally and need no scroll at desktop widths.

**Also permanent, applies to every guide from here on:** a "Save as image" button next to the reference table, exporting a downloadable PNG (~1200px wide) rendered from the live DOM via Canvas 2D/Path2D. Do not use html2canvas (unwanted new dependency) or an SVG-foreignObject approach (risks a tainted-canvas SecurityError with cross-origin Google Fonts). Proven working on ubo-investigation-handbook.html, confirmed on both mobile and desktop.

**Also required:** every new guide's card in `knowledge.html` must carry a `data-date="YYYY-MM-DD"` attribute matching its actual go-live date, wherever it's physically inserted in the HTML, the existing `sortCards()` function handles correct chronological positioning automatically. No manual reordering needed or expected.

This standard was adopted mid-project. Guides already live before this point are not automatically retrofitted, that's a separate scoped decision if it happens. Both Fraud Red Flags and False Positive Playbook are now built against this standard, proving it works in practice.
